Security Mindset


The Security Mindset is a way of thinking which emphasises the fallability or holes in a system.





Reasons for Security

Security needs to be put in place in order to ensure Confidentiality, Integrity and Availabilty "CIA" of data.
Confidentiality refers to the protection of data from unauthorised third parties.
Integrity means data is kept as it is supposed to be and unwanted changes are corrected and/or prevented (i.e. it is not corrupted).
Availability ensures data can be accessed at any time and from any location by authorised people.

Security is neccessary to protect people and assests from potential harm such as financial loss or legal consequences due to data theft or identity theft for example.
Security provides assurance that people's assets as well as legal, professional and social personalities are protected.
It therefore facilitates trust between interacting parties and promotes legal compliance as upholds good governance.





Methods of Security

Security involves implementing and reinforcing the "3 A's" in order to ensure accountability regarding the use of information systems:

Authentication (Biometrics, Passwords, Tokens)
Authorisation (Access control)
Audit (Forensic analysis, Detection of intruders)

Controls

Controls are security measures which are implemented to mitigate risk. There are 3 broad categories of controls . . .
  • Physical
    Infrastructures such as cameras, gates, guards and locks which prevent outsiders coming into contact with valuable assets.

  • Logical
    Technical features such as encryption, firewalls and passwords which prevent unauthorized activities.

  • Administrative
    Laws, policies and procedures which govern interaction

Viruses


Harmful code developed by attackers which causes destruction to the system.
Viiruses can be planted into systems or may spread via email and/or (removable) disks.


Trojan Horses


Malicious programmes or software code disguised inside a safe- or normal-looking programme.
When a user runs the programme, the disguised code runs aas well, deleting files and/or corrupting the computer.
(Usually spread via email attachments)


Worms


Programs which run independently; travelling from computer to computer accross network connections.
Segments of the worms may be running on multiple computers simultaneously. They usually do not change other programs but may transport code which does.


Password Cracking


A technique used by attackers to secretly gain access to a system through a third party user's account.
This is usually achieved through abuse of or theft of trusted or personal information.


Denial of Service (DoS) Attacks


This is an attack in which hackers flood a Web server with communication in order to keep it pre-occupied or 'break' the system which results in loss due to the compromised neccessity of having the service available.


Email Hacking

Impersonation

Impersonation : The sender address on internet email cannot be trusted as the sender can create a false return address.
It is possible for someone to have modified the header in transit, or for the sender to have connected directly to the Simple Mail Transfer Protocol (STMP) port on the target computer to enter the email.

Eavesdropping

Eavesdropping : Email headers and contents get transmitted in plain text if no encryption is used.
This means that the contents of a message can be read or altered in transit.
It is possible to modify the header to hide por change the sender, or to redirect the message.


Packet Replay


Packet Replay : Message packets in a network can be recorded and retransmitted. This is a significant threath for programmes which use authentication sequences as intruders are able to replay legitimatenauthentication sequence messages to gain access to a system. Packet replay is generally almost impossible to detect; however, it can be prevented using packet time-stamping and packet sequence counting.


Social Engineering


Social Engineering : This is a common form of cracking which can be used by those external to or internal to an organisation. It is a method in which hackers coax people into
erroneously revealing their password or confidential security information.


Intrusion Attacks


Intrusion Attacks : Intrusion attacks involve hackers using hacking tools such as password-cracking tools or manipulation tools
in order to gain access to closed systems. Intrusion detection tools can detect changes and variants which take place within a system or network; and have been effective in deterring intruders.


Network Spoofing


Network Spoofing : In network spoofing, a system presents itself to the network as though it were a different system (computer A impersonates computer B by sending B's address instead of its own).
This is done in order for the intruder to gain entry into an otherwise closed group of trusted systems. Trust is imparted in a one-to-one manner: computer A trusts computer B (this does not mean computer B trusts computer A)
Additionally, it is taken for granted that the system administrator of the trusted system is performing its job properly by maintaining an appropriate level of security for the system.
The opportunity arises for network spoofing to work in the following way: computer A trusts computer B and computer C spoofs (impersonates) computer B, allowing computer C to gain access to computer A, a system it would otherwise have been excluded from.
